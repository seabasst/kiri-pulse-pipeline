ARCHITECTURE OVERVIEW

AI-Powered Report Pipeline

Every week, raw advertising data flows through five stages to become a polished Google Slides report — with zero manual work.

1
Meta Ads
Data Source

Facebook and Instagram campaigns run across multiple markets. Every impression, click, and purchase is tracked by Meta's ad platform in real time. This is where all the raw performance data originates.

Campaigns
SE-Brand123-Catalog, NO-Brand123-Evergreen, SE-Brand123-Offers + multi-market
Brands
Brand123 (main) + Brand123-Outlet (retargeting)
Markets
Sweden, Norway, Denmark, UK
Metrics tracked
Spend, Impressions, Clicks, CTR, CPC, Purchases, Revenue, ROAS, CPA
Meta Marketing API → nightly sync to warehouse
2
Google BigQuery
Data Warehouse

A nightly pipeline pulls the previous day's ad data from Meta and loads it into BigQuery. Each row represents one ad on one day. This gives us a clean, queryable history of every campaign across all markets.

Project / Dataset
kiri-media-reporting / brand123
Table
brand123.meta_ads
Row grain
One row per ad per day — keyed on time_increment_day
Market detection
Extracted from campaign name prefix via REGEXP_EXTRACT — e.g. "SE-", "NO-"
5 SQL queries via google.cloud.bigquery Python SDK
3
Claude + Kiri Pulse
Intelligence Layer

Claude reads the client's config file to understand KPIs, warning thresholds, and seasonal strategy. It then runs five SQL queries against BigQuery to pull the current week, the previous week, breakdowns by country, top campaigns, top ads, and underperformers. The results are compared, scored, and turned into actionable insights.

Data Queries

_query_weekly_summary() → 7d vs previous 7d
_query_by_country() → SE / NO / DK / UK split
_query_top_campaigns() → top 5 by ROAS
_query_top_ads() → top 5 by revenue
_query_bottom_campaigns() → underperformers

Anomaly Detection Week-over-Week Comparison Market Breakdown Auto-Recommendations Campaign Scoring Warning Flags
MCP Google Slides API + Drive API
4
Google Slides Design Engine
Presentation

A custom design engine builds each slide programmatically through the Google Slides API. It uses a consistent design system with metric cards, data tables, color-coded ROAS indicators, and branded headers. Every element is positioned in EMU units for pixel-perfect layout.

Layout engine
